## 学习目标

掌握 Linux 性能分析的完整方法体系，能通过工具链快速定位瓶颈，理解 USE 和 RED 方法论，有能力进行系统级性能调优。

## 核心方法论

- **USE 方法**：对每种资源，检查 Utilization（利用率）、Saturation（饱和度）、Errors（错误）
- **RED 方法**：对每个服务，检查 Rate（请求速率）、Errors（错误率）、Duration（延迟）
- **60秒排查法**：uptime → dmesg → vmstat → mpstat → pidstat → iostat → free → sar
