# 02-内核架构

## 内核类型

- **宏内核 (Monolithic)** — Linux 采用。全部内核功能运行在同一地址空间，效率高但模块耦合度高。
- **微内核 (Microkernel)** — Minix、QNX。最小化内核功能，其余在用户态运行。更安全但 IPC 开销大。
- **混合内核 (Hybrid)** — Windows/macOS。折中方案。

## Linux 内核子系统

```
┌─────────────────────────────────────────┐
│            System Call Interface         │

├──────┴──────┴──────┴──────┴──────┴──────┤
│           Architecture-Dependent         │
└─────────────────────────────────────────┘
```

## 关键子系统

### 进程调度器 (Process Scheduler)
- **CFS (Completely Fair Scheduler)** — 默认调度器，红黑树维护 vruntime
- **实时调度类** — SCHED_FIFO / SCHED_RR
- **Nice 值范围**：-20（最高优先级）到 +19（最低）

### 内存管理 (Memory Management)
- 虚拟地址空间、页表、TLB
- Buddy System 分配物理页
- Slab 分配器管理内核对象
- 页面回收（kswapd）与 OOM Killer

### 虚拟文件系统 (VFS)
- 统一的文件操作接口（open/read/write/close）
- 支持 ext4、xfs、btrfs、nfs 等多种具体文件系统
- Page Cache：缓存文件内容，加速 IO

### 网络协议栈
- 分层实现：Socket → TCP/UDP → IP → 设备驱动
- Netfilter 框架（iptables 的基础）
- 高性能数据路径：XDP/eBPF

## 内核版本

- `uname -r` 查看当前版本
- 主线版本 → 稳定版 → 长期支持版（LTS）
- SRE 应关注：内核参数调优（sysctl）、安全更新、eBPF 新特性

## 关键命令

```bash
uname -a          # 内核版本信息
cat /proc/version  # 详细版本
lsmod             # 已加载内核模块
modinfo <module>  # 模块详情
```
