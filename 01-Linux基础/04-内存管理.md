# 04-内存管理

## 内存层次结构

```
CPU 寄存器 (最快, ~1ns)
  → L1 Cache (~1ns, 32KB)
    → L2 Cache (~4ns, 256KB)
      → L3 Cache (~12ns, 8-32MB)
        → 主存 RAM (~100ns, GB级别)
          → SSD (~100μs, TB级别)
            → HDD (~10ms, TB级别)
```

## 虚拟内存

### 核心概念

每个进程拥有独立的虚拟地址空间，通过 MMU（内存管理单元）将虚拟地址翻译为物理地址。

```
虚拟地址 → 页表(Page Table) → 物理地址
                ↓ (miss)
            TLB (Translation Lookaside Buffer)
```

### 进程内存布局

```
高地址 ┌──────────────┐
      │   Kernel Space  │ (不可直接访问)
      ├──────────────┤
      │   Stack         │ ↓ 向下增长
      │   ...           │
      │   mmap region   │ (共享库、文件映射)
      │   ...           │
      │   Heap          │ ↑ 向上增长 (brk/sbrk)
      │   BSS           │ (未初始化数据)
      │   Data          │ (已初始化数据)
低地址 └──────────────┘ Text (代码段)
```

## 内存分配机制

### Buddy System
- 将物理内存按 2^n 页框组织
- 分配时找到合适大小块，必要时拆分
- 释放时尝试合并相邻空闲块

### Slab 分配器
- 缓存常用内核对象（inode、dentry、task_struct）
- 减少反复分配/释放的开销
- 查看：`cat /proc/slabinfo`

## 页面回收与交换

### kswapd 守护进程
- 当空闲内存低于阈值时唤醒
- 回收 Page Cache、匿名页
- 三个水位线：WMARK_HIGH / WMARK_LOW / WMARK_MIN

### Swap
- 将不常用的匿名页换出到磁盘
- `swappiness` 参数（0-100）控制倾向
- 数据库场景建议设为低值（1-10）

## OOM Killer

内存耗尽时内核的选择机制：
- 每个进程有 `oom_score`（0-1000，越高越可能被杀）
- `oom_score_adj` 可以手动调整（-1000 豁免，1000 优先杀）
- 优先杀子进程，尽量减少损失

## 关键命令

```bash
free -h              # 内存概览
vmstat 1             # 内存活动统计
cat /proc/meminfo    # 详细内存信息
pmap -x <PID>        # 进程内存映射
smem -p              # PSS/USS/RSS 分析
```

## /proc/meminfo 关键字段

| 字段 | 含义 |
|------|------|
| MemTotal | 总物理内存 |
| MemAvailable | 可用内存（含可回收缓存） |
| Buffers | 块设备缓冲 |
| Cached | Page Cache |
| SwapTotal/SwapFree | 交换空间 |
