# 03-故障转移设计

## 故障检测

| 方法 | 原理 | 响应速度 |
|------|------|----------|
| 心跳 (Heartbeat) | 定期 ping | 秒级 |
| 外部探测 | LB 健康检查 | 秒级 |
| Gossip 协议 | 节点间相互探测 | 秒-分级 |
| 租约 (Lease) | 过期未续约即判定故障 | 分钟级 |

## 故障转移模式

### 冷备 (Cold Standby)
- 备节点不启动，故障时手动启动
- 恢复时间：分钟-小时
- 成本最低

### 温备 (Warm Standby)
- 备节点运行但不同步流量
- 恢复时间：分钟级
- 常见方案：Keepalived + VIP

### 热备 (Hot Standby)
- 备节点同步运行，可随时接替
- 恢复时间：秒级
- 常见方案：多活 + 自动故障转移

## Keepalived 示例

```bash
# /etc/keepalived/keepalived.conf
vrrp_instance VI_1 {
    state MASTER           # 或 BACKUP
    interface eth0
    virtual_router_id 51
    priority 100           # MASTER 优先级更高
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass secret
    }
    virtual_ipaddress {
        10.0.0.100/24      # VIP
    }
    track_script {
        chk_nginx          # 跟踪 Nginx 健康
    }
}
```

## 脑裂 (Split-Brain)

两个节点都认为自己是主：
- **预防**：法定人数 (quorum) > N/2
- **检测**：共享存储 fencing
- **恢复**：STONITH (Shoot The Other Node In The Head)

## 降级策略

当部分功能不可用时，优雅降级：
- **静态页面** — CDN 兜底
- **限流** — 保护核心功能
- **功能开关** — 紧急关闭非核心功能
